= Troubleshooting CoreDNS with the `kuadrant` plugin

[role="_abstract"]
tbd text here

You can view CoreDNS logs by running the following command:
[source, bash]
----
$ kubectl logs -f deployments/kuadrant-coredns -n kuadrant-coredns
----

You can get recent logs by running the following command:
[source, bash]
----
$ kubectl logs --tail=100 deployments/kuadrant-coredns -n kuadrant-coredns
----

//what problems might be solved if we forgot to restart coredns after a change?

After updating the ConfigMap through the DNS Operator, restart CoreDNS by using the following command:
[source,terminal]
----
$ oc -n kuadrant-coredns rollout restart deployment kuadrant-coredns
----
